# 관리자 이미지 업로드 가이드

## 개요

- 관리자 콘텐츠 등록/수정 화면에서 콘텐츠 썸네일 이미지를 직접 업로드할 수 있습니다.
- 관리자 배너 등록/수정 화면에서 배너 이미지를 직접 업로드할 수 있습니다.
- 업로드 전 브라우저에서 먼저 리사이즈와 압축을 수행해 무료 플랜 사용량을 줄입니다.
- 업로드 API는 관리자 세션을 확인한 뒤 서버에서 처리하며, `SUPABASE_SERVICE_ROLE_KEY`는 클라이언트에 노출되지 않습니다.

## 필수 환경변수

`.env.local`에 아래 값이 필요합니다.

```env
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
```

## Supabase Storage 설정

1. Supabase 대시보드에서 `Storage`로 이동합니다.
2. 아래 두 개의 public 버킷을 생성합니다.
3. `thumbnails`
4. `banners`
5. 브라우저가 Storage에 직접 업로드하지 않고, 관리자 인증을 통과한 서버 API가 service role로 업로드를 처리합니다.

## 업로드 정책

### 콘텐츠 썸네일

- 버킷: `thumbnails`
- 경로 형식: `contents/YYYY/MM/<uuid>.<ext>`
- 허용 형식: `jpg`, `jpeg`, `png`, `webp`
- 원본 최대 용량: 10MB
- 긴 변 최대 크기: 1600px
- 목표 용량: 약 450KB 이하
- 권장 비율: `16:9`
- 권장 해상도: `1600x900px 이상`
- 안전 영역 가이드: 텍스트, 얼굴, 로고 같은 핵심 요소는 가장자리에서 10% 이상 안쪽에 배치합니다.

### 배너 이미지

- 버킷: `banners`
- 경로 형식: `promotions/YYYY/MM/<uuid>.<ext>`
- 허용 형식: `jpg`, `jpeg`, `png`, `webp`
- 원본 최대 용량: 10MB
- 긴 변 최대 크기: 1800px
- 목표 용량: 약 550KB 이하
- 배너는 콘텐츠 썸네일과 별도로 운영하며, 현재 퍼블릭 노출 비율은 `16:7`입니다.

## 퍼블릭 렌더링 원칙

- 콘텐츠 썸네일은 카드, 리스트, 히어로, 소형 카드, 상세 상단 이미지를 모두 `16:9` 기준으로 맞춥니다.
- 배너는 콘텐츠 썸네일과 구분된 별도 비율 전략을 유지합니다.
- 퍼블릭 페이지의 원격 이미지는 기본 `<img>`로 렌더링해 Vercel 이미지 최적화 비용을 늘리지 않습니다.

## 실패 처리

- 새 썸네일 업로드 뒤 콘텐츠 저장이 실패하면 방금 올린 파일은 정리 대상이 됩니다.
- 썸네일 교체 저장이 성공하면 이전 Storage 파일은 자동 정리됩니다.
- 배너도 같은 원칙으로 교체/삭제 시 기존 파일을 정리합니다.

## 운영 체크 포인트

- 관리자 로그인 세션이 정상이어야 업로드 API가 `401` 없이 동작합니다.
- `NEXT_PUBLIC_SUPABASE_URL`과 배포 환경값이 동일하게 맞춰져 있어야 합니다.
- public 버킷이므로 민감한 이미지는 올리지 않습니다.
- 업로드 직후 페이지를 닫으면 임시 파일이 남을 수 있으니 저장 완료까지 기다립니다.
