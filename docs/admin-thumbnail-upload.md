# 관리자 이미지 업로드 가이드

## 개요

- 관리자 콘텐츠 등록/수정 화면에서는 썸네일 이미지를 직접 업로드할 수 있습니다.
- 관리자 배너 등록/수정 화면에서는 배너 이미지를 직접 업로드할 수 있습니다.
- 업로드 전 브라우저에서 먼저 리사이징과 압축을 수행해 무료 플랜 사용량을 줄입니다.
- 업로드는 관리자 로그인 세션을 확인하는 서버 API가 처리하며, `SUPABASE_SERVICE_ROLE_KEY`는 클라이언트에 노출되지 않습니다.

## 필요한 환경변수

`.env.local`에 아래 값이 필요합니다.

```env
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...
```

## Supabase Storage 설정

1. Supabase 대시보드에서 `Storage`로 이동합니다.
2. 아래 두 개의 public 버킷을 생성합니다.
3. `thumbnails`
4. `banners`
5. 이번 구현에서는 브라우저가 Storage에 직접 쓰지 않습니다.
6. 관리자 인증을 통과한 서버 API가 service role로 업로드와 삭제를 처리합니다.

## 업로드 정책

### 콘텐츠 썸네일

- 버킷: `thumbnails`
- 경로 형식: `contents/YYYY/MM/<uuid>.<ext>`
- 허용 형식: `jpg`, `jpeg`, `png`, `webp`
- 원본 최대 용량: 10MB
- 긴 변 최대 크기: 1600px
- 목표 용량: 약 450KB 이하

### 배너 이미지

- 버킷: `banners`
- 경로 형식: `promotions/YYYY/MM/<uuid>.<ext>`
- 허용 형식: `jpg`, `jpeg`, `png`, `webp`
- 원본 최대 용량: 10MB
- 긴 변 최대 크기: 1800px
- 목표 용량: 약 550KB 이하

## 퍼블릭 이미지 렌더링 정책

- 퍼블릭 페이지의 콘텐츠 썸네일과 배너 이미지는 원격 이미지일 때 브라우저 기본 `<img>`로 렌더링합니다.
- 이 방식은 Vercel의 원격 이미지 최적화 비용을 유발하지 않아 무료 플랜에 더 안전합니다.
- 로컬 정적 이미지가 필요할 때만 `next/image`를 사용합니다.
- 업로드 단계에서 이미 충분히 압축해 두므로 시각 품질 저하를 최소화할 수 있습니다.

## 정합성 처리

- 새 썸네일 업로드 후 콘텐츠 저장이 실패하면 방금 올린 파일은 자동 삭제됩니다.
- 썸네일 교체 저장이 성공하면 이전 Storage 파일도 자동 삭제됩니다.
- 새 배너 업로드 후 배너 저장이 실패하면 방금 올린 파일은 자동 삭제됩니다.
- 배너 교체 저장이 성공하면 이전 Storage 파일도 자동 삭제됩니다.
- 배너 삭제 시 연결된 Storage 파일도 함께 정리합니다.

## 운영 전 확인 사항

- 관리자 로그인 세션이 정상 동작해야 업로드 API가 401 없이 동작합니다.
- `NEXT_PUBLIC_SUPABASE_URL`은 배포 환경에도 동일하게 설정되어 있어야 합니다.
- public 버킷이므로 민감한 이미지는 올리지 않는 운영 원칙이 필요합니다.
- 저장하지 않고 관리자 페이지를 이탈하면 직전에 업로드한 임시 파일이 남을 수 있습니다.
